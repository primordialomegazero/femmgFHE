  I-represent ang ciphertexts sa group

ANG IDEA:
---------
Sa halip na polynomials (RLWE), gumamit ng group elements
kung saan ang "multiplication" ay group operation (walang noise)

POSSIBLE GROUPS:
----------------
1. Matrix groups (GL(n, q))
   - Multiplication = matrix multiplication
   - May inverse, identity
   - Pwedeng i-homomorphically compute

2. Elliptic curve groups
   - Addition = group operation
   - Pwedeng i-homomorphically compute

3. Symmetric groups (S_n)
   - Permutation composition
   - Non-commutative, mahirap i-invert

CHALLENGES:
-----------
1. Security: Kailangan ng hard problem (e.g., discrete log)
2. Homomorphism: Kailangan ng operation na "preserves" structure
3. Bootstrapping: Hindi kailangan if noise-free

STATUS:
-------
- Theoretical pa lang
- Walang practical implementation
- Kailangan ng new mathematical framework
